Java 8 parallelStream 在大批量小任务上的性能陷阱

tl;dr

Java 8 的 parallelStream 在运行大量的短任务（短时间执行的任务）之后会变成单线程运行，这是 parallelStream 自带的性能优化功能。但是在短任务和长任务交替执行时，这种优化会显著拖慢运行速度。
